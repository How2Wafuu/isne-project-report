\begin{abstractTH}
โครงการนี้เสนอสถาปัตยกรรมระบบการจัดการข้อมูลและเหตุการณ์ด้านความปลอดภัย (SIEM) ที่คุ้มค่า ซึ่งสร้างขึ้นบนแพลตฟอร์มโอเพนซอร์ส Wazuh เพื่อทำการตรวจสอบมาตรฐานความปลอดภัยทางไซเบอร์ HAIT+ (Healthcare Accreditation Information Technology) ในโรงพยาบาลระดับอำเภอที่มีทรัพยากรจำกัดโดยอัตโนมัติ ระบบนี้รวมศูนย์การรวบรวมบันทึกข้อมูลจากโครงสร้างพื้นฐานที่สำคัญ รวมถึงฐานข้อมูลทางการแพทย์บนระบบ Linux (MariaDB) และเวิร์กสเตชัน Windows ส่วนสำคัญทางด้านวิศวกรรมคือไปป์ไลน์ “Command-to-Rule” ซึ่งใช้เอเจนต์ปลายทางในการรวบรวมเมตริกการดำเนินงานที่ไม่ได้รับการติดตามโดยตรง และแยกวิเคราะห์เมตริกเหล่านั้นผ่านกฎ XML ที่กำหนดเองสำหรับการแจ้งเตือนการปฏิบัติตามข้อกำหนด ความท้าทายในการใช้งาน เช่น การชนกันของตัวถอดรหัส MariaDB และรหัสเหตุการณ์ Windows ที่ไม่น่าเชื่อถือ ได้รับการแก้ไขโดยใช้ลำดับชั้นของกฎและการตรวจสอบการสร้างกระบวนการ (เหตุการณ์ 4688) การตรวจสอบความถูกต้องผ่านการจำลองการโจมตีแสดงให้เห็นอัตราความครอบคลุมการปฏิบัติตามข้อกำหนด\textbf{โดยตรง 60\%} (15 จาก 25 ตัวชี้วัด HAIT+) ส่วนที่เหลืออีก 40\% จำเป็นต้องมีการบูรณาการกับระบบภายนอกในอนาคต เช่น ระบบตรวจจับการบุกรุกเครือข่าย (NIDS) และฐานข้อมูลด้านทรัพยากรบุคคล
\end{abstractTH}

\begin{abstract}
This project presents a cost-effective Security Information and Event Management (SIEM) architecture built on the open-source Wazuh platform to automate monitoring for the HAIT+ (Healthcare Accreditation Information Technology) cybersecurity standard in resource constrained district hospitals. The system centralizes log collection across critical infrastructure, including Linux-based medical databases (MariaDB) and Windows workstations. A key engineering contribution is the “Command-to-Rule” pipeline, which uses endpoint agents to gather operational metrics not natively tracked, parsing them via custom XML rules for compliance alerting. Implementation challenges, such as MariaDB decoder collisions and unreliable Windows Event IDs, were resolved using Rule Hierarchy and Process Creation auditing (Event 4688). Validation through attack simulations demonstrated a \textbf{60\% direct compliance coverage rate} (15 out of 25 HAIT+ indicators). The remaining 40\% requires future integration with external systems, such as Network Intrusion Detection Systems (NIDS) and HR databases.
\end{abstract}
